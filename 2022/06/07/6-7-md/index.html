ch5 XOR

Arithemetic:

  1. x XOR x = 0
  2. x XOR 0 = x
  3. x XOR y = y XOR x
  4. (x XOR y) XOR z = x XOR (y XOR z)
  • Generally x XOR y XOR = y since
    x XOR y XOR x = x XOR x XOR y
    = 0 XOR y
    = y

bitwise exmple in python

73^87 outputs 30 since
73 = 0b1001001, 87 = 0b1010111,where 0b means numbers are stored as binary
XOR bitwisely, getting 0b0011110 = 30

XOR as a one-time pad

To be secure

  1. pad can be used only once
  2. pad has to be truly random

    attack when pad used more than once

  3. c1 XOR c2 = (p1 XOR k) XOR (p2 XOR k) = … = p1 XOR p2
    looks fine but
  4. ci XOR pi = (pi XOR k) XOR pi
    = pi XOR pi XOR k
    = k
    key leak, even a only leaking ki(a bit of entire key) can be fatal.

  • iteratively, even multi-pad pads are broken due to NPL and ML

  • Keyexchange and secruity are vital in practice.

    Ch6 block cipher

    attack on AES-ECB:

  1. send a plaintext A of length 1 byte shorter than blocksize, the block will be (A||s0),s0 is first byte. E(A||s0) = Cr1 is known to attacker.
  2. send a plain text A’ of block size,brute force s0, as it can verified with Cr1.
  3. Iteratively repeat, brute force each byte of secret can be done in p*b attempts where p is # of possible values in each byte, typically 2^8 =256, b is blocksize
