ch5 XOR

Arithemetic:

  1. x XOR x = 0
  2. x XOR 0 = x
  3. x XOR y = y XOR x
  4. (x XOR y) XOR z = x XOR (y XOR z)
  • Generally x XOR y XOR = y since
    x XOR y XOR x = x XOR x XOR y
    = 0 XOR y
    = y

bitwise exmple in python

73^87 outputs 30 since
73 = 0b1001001, 87 = 0b1010111,where 0b means numbers are stored as binary
XOR bitwisely, getting 0b0011110 = 30

XOR as a one-time pad

To be secure

  1. pad can be used only once
  2. pad has to be truly random

    attack when pad used more than once

  3. c1 XOR c2 = (p1 XOR k) XOR (p2 XOR k) = … = p1 XOR p2
    looks fine but
  4. ci XOR pi = (pi XOR k) XOR pi
    = pi XOR pi XOR k
    = k
    key leak, even a only leaking ki(a bit of entire key) can be fatal.

  • iteratively, even multi-pad pads are broken due to NPL and ML

  • Keyexchange and secruity are vital in practice.

    Ch6 block cipher

    attack on AES-ECB:

  1. send a plaintext A of length 1 byte shorter than blocksize, the block will be (A||s0),s0 is first byte. E(A||s0) = Cr1 is known to attacker.
  2. send a plain text A’ of block size,brute force s0, as it can verified with Cr1.
  3. Iteratively repeat, brute force each byte of secret can be done in p*b attempts where p is # of possible values in each byte, typically 2^8 =256, b is blocksize

    ch7 stream cipher

    encryption only prevenets attacker from reading , but it is not enough without authentification.

C1.4 Authenticated encryption

AE: communicate with known key and symmetric encryption

AES: advacned encription standard and its developments

  • security: mostly AES-128 ,require 2*128 opertions to brute force

  • general procesure:

    1. take a variable-length key
    2. takes a plain-text of 128bits(usually)
    3. output s a cyphertext of 128bits

  • a little detaied for enryption:

    1. transform input into square matrix
    2. apply subbytes, shiftrows, mixclomuns Funtion to input.
      note: above 3 operations are inversible
    3. apply AddRoundKey funciont using XOR operation to get output cyphertext

  • To transform into square matrix, padding is necessary, aka mode of operation.

  • Naive way of direclty (ECB,electronic codebook)dividing into boocks of 16bytes will lead to info leak of pattern as encryption is deterministic, typical example

  • To overcome , CBC(cypher block chaining) use a unpredictable and random IV(initialization vector) to randomize input before encryption.

  • to encrypt with CBC operation, generate a random IV and XOR with first 16bytes of plaintext before encryption.

  • WARNING: IV need to be unique and unpredictable!

  • Use previous cyphertext to XOR with next block of input iteratively.

  • To add integrity to the ciphertext , add addtional HMAC as authentication tag, forming a AES-CBC-HMAC construction.

    From AES-CBC-HMAC to AEAD(AE with associated data)

  • To be developer-friendly and standarlized, AEAD is introduced

    AES-GCM(AES with Galois Counter Mode) AEAD

  • It apply AES with a key to a 128bit string consisting of a nonce(a random number with max lenght 96 bits) concatenated with a count number(max 32bits)to produce a keystream

  • AES-GCM avoids padding as it can truncates the length of keystream to fit with length of input

AEAD alternative: ChaCha20-Poly1305

  1. Speed:
    • Recall: AES is fast with hardware support
    • In contrast, chacha20 and poly1305 were designed to be fast with software
  2. chacha20 takes a key and a unique nonce to produce many blocks of 64bits keystream by calling block function, which takes a 256 bits key, a 92 bits nonce, a 32 bits counter as in AES and AES-GCM.
    Then XOR the keystrem with plaintext to produce ciphertext.

MAC message authentication codes

Chapter1.2 Hash: length of output(digest) is fiexed

min length: (currently) 128 bits for property 1,2 256bits for collision

1. 3 vital properties:

  1. pre-image resistance:
    Given (public algorithm F) and output y
    It should not be possible to reverse the F and find out input x

  2. second pre-iamge resistance:
    Given (public Algorithm F) , output y, input x1
    It should not be possible to find another
    input x2 s.t. F(x2) = y

  3. collision resistance:
    Given (public Algorithm F)
    It should not be possible to construct x1!=x2 s.t. F(x1) = F(x2)

2. Usage: openssl cli

ex: use sha256 to hash “hello”

result:

3. Warning: sha1 ,M5 is broken ; sha2 is vulnerable due to length-extension attacks

4. XOR: exclusive or

1 xor 0 = 1
1 xor 1 = 0 !!!
0 xor 1 = 1
0 xor 0 = 0

5. Compression function:

Taking two inputs, produce one output ,using XOR operator.

6.Construction: Merkle-Damgard/Sponge

Sha2: Merkle-Damgard

  1. Add padding to input to fit certain length s.t. it can be divided into chunks of same block size to fit in argument of compression function.
  2. Iteratively applies compreesion function to msg, use previous ouput as 2nd argument for compression function, add padding whereever necessary as in step1.
  • (vulnerable to length-extension attack)

Sha-3: Sponge

  • keccak-f permutation
  • use permutation to map all possible inputs and possible ouputs
  • separate input lenght = rate + capacity(secret)
  • initiate with 0s, absorb input by XOR with input length, repeat similarly as above
